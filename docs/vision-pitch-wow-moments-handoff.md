### 0.5.1 Wow 1 阶段沉淀的跨 Wow Moment 设计决策（下一个 agent 处理 Wow 2/3 时**必须继承**）

**决策 1 · 空间变化的两层分类**（细节层 vs 结构级）
- Wow 1 采用"段 3-4 累积细节层，段 5 集中触发结构级"的节奏，保护"真名揭示"作为最长单一 wow moment 的对比冲击力
- **对 Wow 3 的含义**：原创英灵召唤的空间蜕变也应遵循此节奏——初遇灰模态 → 逐次亲历累积细节层 → 真名/身份揭示时集中触发结构级质变。避免 Wow 3 的空间演化被压缩成"一次性爆发"
- **细节层清单示例**：NPC 从模糊变清晰 / 物件被命名 / 局部饱和度提升 / 环境音层次叠加
- **结构级清单示例**：封闭大门打开 / 地平线首次浮现 / 光源方向明确 / 主题旋律进入 / 色饱和度整体跳升 / 玩家与英灵视线首次交汇 / 大件物（地图桌等）从无到有
- **红线**：结构级变化在真名揭示前**绝不能提前触发任何一项**（会稀释质变冲击）

**决策 2 · 意外触发式开场**（Wow 1 采用："搬家 → 储藏室 → 物品滑落到灰尘覆盖的法阵上"）
- 这个桥段锁死了"这不是道具组预设"的证据感，同时为"英灵之座 / 灵基空间的显影"设定层提供合法性（法阵早已存在，不是玩家画的）
- **对 Wow 3 的含义**：是否沿用意外触发风格待下游决定
  - 如果沿用：需要设计不同的意外情境（例如"整理爷爷遗物时无意翻出怀表"），避免与 Wow 1 结构重复
  - 如果不沿用：需要设计 Wow 3 特有的召唤入口，但仍需保留"物品普通到不起眼"与"法阵 / 仪式的被遗忘感"两个核心约束

**决策 3 · 触媒物品选择的三条原则**（Wow 1 主案 = 一本泛黄的旧书）
- **普通到不起眼**（G1 证据感的基础）
- **与英灵有可事后追溯的深层关联**（G3 因果归因的基础）
- **掉落 / 呈现时有戏剧性重量感**（观感的基础）
- **对 Wow 3 的含义**：原创英灵触媒（handoff 中示范物品"爷爷的怀表"）符合前两条，掉落感需要下游确认；如果沿用意外触发桥段，怀表的掉落感偏弱，可能需要备选

**决策 4 · 玩家 avatar 全程不露面部**（跨 Wow Moment 通用约束）
- 只用背影 / 手部 / 局部；服装 / 体型中性化；avatar 是"观众的化身"而非"游戏中的角色"
- **对 Wow 2 的含义**："上线并肩作战"段容易有露脸镜头的冲动（战斗后擦汗 / 摘耳机的表情特写），务必控制——即使段 5 现实收尾"Master 摘下耳机靠椅背笑了"，也用背影或侧局部呈现，不用正脸
- **对 Wow 3 的含义**：召唤仪式的观众代入感依赖玩家保持中性

**决策 5 · 玩家反应克制**（跨 Wow Moment 语言约束）
- 惊愕但不夸张、不预演、不表演。不做戏剧化的"呐喊 / 挥拳 / 眼泪特写"
- **对 Wow 2 的含义**：并肩作战段玩家的"我做得到"要冷静而非热血；击杀 Rider 后 Master 站在原地喘气而非欢呼
- **对 Wow 3 的含义**：一战工兵敬礼时玩家的反应用"手扶桌面沉默片刻"或"目光凝视片刻"，不做感慨性的语言

## 1. 核心命题（tagline · 不可改动）

> **和现实世界的时间同步，在另一个世界中时刻在进行着的真实的圣杯战争。玩家要召唤英灵、与英灵建立羁绊、找到最佳的与之配合的方式和策略来赢下圣杯。**

这一句是 vision pitch 的收尾字幕候选。**它同时锁定了 4 件事**——任何具体内容设计不得违反这 4 条：

1. **实时同步** —— 与现实世界时间同步，拒绝回合制、拒绝"随时暂停"式假流转
2. **另一个世界时刻在进行** —— 玩家在场或不在场都在推进。这是"多个世界"愿景的新解读：不是空间维度的多星球，是**时间维度上的另一个世界在时刻推进**
3. **真实** —— 反预渲染、反脚本、可证明这是 AI 现场生成的
4. **游戏循环** —— 召唤 → 建立羁绊 → 找到最佳配合方式 → 赢圣杯

## 1.5 整体时长约束（不可突破 · 关键）

**Vision pitch demo 视频最多 90 秒**。这是 target render / sizzle reel 的常规体量，不是可玩体验时长。

**推荐 90 秒时长分配**（下一个 agent 可微调段内秒数，但**不得突破 90 秒总长**）：

| 段 | 秒数 | 累计 | 备注 |
|---|---|---|---|
| **Wow 1** · 召唤 → 空间灰模 → 记忆亲历闪回 → 真名揭示 · 空间质变 | **30-35 秒** | 35 秒 | 三段中弧线最完整、占最多时长 |
| **Wow 2** · 早上约定 → 现实一天蒙太奇 → 上线并肩作战 → 下线交接 | **30-35 秒** | 70 秒 | 主戏在上线并肩作战段（约 15-20 秒） |
| **Wow 3** · 原创英灵召唤 → 一天时间线闪回 → 收尾字幕 → tagline | **20-25 秒** | 90 秒 | 情感峰值收尾 |

**关键区分**（不要混淆）：
- 本次 vision pitch demo **视频**时长 = **90 秒**（本文档聚焦）
- 未来产品的**可玩体验单元**时长 = 约 **15 分钟**（见 `project-background.md` 第 4 章"愿景碎片"—— 属于产品设计范畴，**本次 pitch 不涉及**）

**为什么最多 90 秒**（不给下一个 agent 争取时长的空间）：
- Target render / sizzle reel 类的行业惯例本就是 60-120 秒
- 内容越短，"这不是剪辑拼凑的"的说服力越强（时间越短越难藏 cut）
- 董事/投资人的注意力预算有限
- 90 秒能演清楚三个 Wow Moment 已经紧张——不是宽松

## 1.6 内容重点优先级（用户显性判定 · 严格执行）

用户已明确 demo 亮点的重点分组。**下一个 agent 的镜头预算必须按此优先级分配**：

### 主要展示（占用绝大部分镜头预算）

- **A · 世界创造 · 英灵与据点空间一体化生成**（对应 **Wow 1 + Wow 3**）
  - 圣遗物召唤（现场生成英灵与其据点空间）
  - 记忆亲历（Animus 内嵌）
  - 真名揭示时的空间质变
  - 原创英灵召唤（Wow 3 证据）

- **B · 英灵 · 有独立意识的战友**（对应 **Wow 2**）
  - 24/7 自主行动
  - 主动联络（战况通信性质，非陪聊）
  - 玩家上线 + 与英灵并肩作战

### 设定支撑（不占用显性镜头预算 · 只通过暗示 / 字幕 / 设定内合法性带过）

- **C · 玩家 agency 与英灵的双向绑定** —— 探索空间 ↔ 了解真名 / 经营空间 ↔ Servant 状态。**在 Wow 1 蜕变过程里隐性承载**，不做单独教学式演出（比如不要拍一个"玩家供品 → 英灵战意值 +1"的过程性镜头）
- **D · 空间-战场双向联通 + 潜入/入侵玩法接口** —— 保留设定内合法性，本 pitch **不显性演出**
- **E · 单次一世界 · 多场次多世界** —— 通过 **Wow 1（知名英灵的空间形态，如马其顿军营）与 Wow 3（原创英灵的空间形态，如一战战壕）的对比**暗示，不另演"多个玩家的世界并列"这类拉远镜头

### 对下一个 agent 的实操含义

- 90 秒时长预算严格按第 1.5 章表格分配：Wow 1 约 30-35 秒服务 A / Wow 2 约 30-35 秒服务 B / Wow 3 约 20-25 秒服务 A 的原创英灵证据 + 收尾
- **看到"C / D / E 需要演出"的冲动出现时——停下来**。它们是设定层的支撑，不是画面预算的消耗者
- **"多个世界"的兑现方式已定**：Wow 1 + Wow 3 的两个截然不同的英灵空间形态对比。**不要再演"拉远看多个玩家世界"这种镜头**（会陷入数量话术 + 消耗宝贵的 90 秒）
- **玩家 agency（C）的存在感通过"玩家和英灵之间的互动质感"隐性传达**，不需要显性展示玩家操作面板 / 经营选项 / 数值变化

## 2. 硬约束继承清单（对下一个 agent 的红线）

### 2.1 从 project-background.md 继承的 6 条设计规则

1. Demo 必须包含 **3 个必现证据时刻同时在场**：
   - ① 个人种子 → 世界在眼前生成
   - ② AI agent 对无法预演的输入作出活反应
   - ③ 拉远看到不止这一个世界
2. 演示物件不是 trailer（cut 来 cut 去、堆酷镜头），是**一段连续的、有人在玩/在操作的演示**——因为要证明"这是真的，不是剪的"
3. Demo 完成后，观众必须能用一句话讲清项目要做什么
4. **AI 标签只出现在通过 4 条件 AND-gate 的动作上**（自然对话 + 世界记得你彩蛋），其他一切走非 AI 范式
5. **绝不承诺 demo 里没演的东西**——每一句演示词、每一帧画面都是债，18 个月内要能兑现
6. **回避数量话术**——"1000 颗星球/万个 NPC/无限内容"出现即扣分；改押"每一个都有意义的差异化"

### 2.2 AI 必要性 4 条件 AND-gate（AI 露出前必须核对）

任何要打 AI 标签的动作必须**同时通过**：
- **G1**：玩家输入空间**不可枚举**（自由文本/语音/自定义素材）
- **G2**：输出在 **demo 时间窗内可观察**（原始 gate 定义为 5–15 分钟；**本次 pitch 视频 90 秒内可观察**——比原始 gate 更严格）
- **G3**：因果链**可被肉眼归因**（评审能看出"我做的事 → 这个结果"）
- **G4**：**单次可靠性够**（评审 probe 下不 OOC / 不翻车）

### 2.3 从 fate-series-research-report.md 继承的 Refuted 层约束

**LLM Agent 目前不可靠支持以下 4 种能力（4 条 claim 均 3-0 反驳）**：
- 多 agent 策略性欺骗
- 涌现非脚本沟通
- 权衡真话与谎言
- 中途动态调整策略

**含义**：任何依赖"AI 涌现 + 多 agent 复杂博弈可靠运作"的设计要格外小心。**Demo 一次说服性演示可行**（G4 单次通过），**长时/多局稳定运作不可承诺**。设计上要**用明确的游戏机制驱动**（如令咒触发条件、真名推理规则）而不是**依赖 LLM 涌现**。

### 2.4 从对话讨论中提炼的红线

在具体内容设计时**必须遵守**（这几条是过程中反复试错换来的经验，不要重蹈覆辙）：

1. **不通过对白展现人设** —— 性格通过**行动**、**决策**、**沟通频率**、**反应模式**传达，不通过独白台词
2. **不演玩家学习曲线的挫败感** —— vision pitch 演的是**可欲望的稳态**，不是学习曲线的痛。任何"玩家先做错再学习"的段落都会被投资人解读为"游戏体验差"
3. **离线消息只做约定和衔接** —— 消息不能成为核心画面（否则被贴"Character.ai / 乙游"标签）。主戏必须是**上线后玩家和英灵在游戏里的行动**
4. **不做多个英灵的差异化对比蒙太奇** —— 一个演示就够。**差异化通过"Wow 1 + Wow 3 呈现两个空间形态截然不同的英灵" + tagline 提示** 隐性传达，不需要显性演出
5. **不承诺规模数量** —— 不出现"7 个英灵同时博弈"、"6 个 Master 同时在战场"这类具体数字。Refuted 层约束要求 demo 只演一次说服性场景
6. **AI 生成的证据只在 Wow 3 显性演出** —— Wow 1 用知名角色（观众有辨识度、能立即识别"AI 演对了"），Wow 3 用原创英灵（AI 凭空生成能力的直接证据）。**顺序不能颠倒**

## 3. 产品核心机制概要（服务于 Wow Moment，不是玩法机制设计）

以下 5 条是 vision pitch 里必须让观众理解的产品概念（不是玩法机制的完整设计，不做深入）：

1. **圣遗物召唤**：玩家用现实物品（拍照 + 描述）作为圣遗物召唤专属英灵。触媒的深层关联决定召唤谁。
2. **据点空间**（灵基空间）：召唤英灵后，一个与英灵背景相关的空间同时建立。**空间既是英灵的居所，也是玩家的探索/建设/共同栖居的场所**。
3. **Agent 英灵**：英灵是有独立性格、独立判断力、24/7 持续行动的 agent。**玩家不在时英灵在圣杯战争里继续推进**。
4. **现实-游戏时间同步**：英灵通过日常终端（手机、电脑）与玩家保持联络。玩家的现实生活时间 = 圣杯战争的时间。
5. **双向联通**：据点空间与圣杯战争公共战场双向联通。**只有 Master 和其英灵可自由出入**，其他人（含其他 Master / Servant）进入需 Master 许可。**预留潜入 / 入侵敌方据点空间的玩法接口**（Assassin 类的天然表达）。

**未决/暂挂**（可留给下一个 agent 或后续会议）：
- 空间持久性：永久 / 契约期间 / 半持久（Servant 消失后是否留"痕迹版"）
- 圣杯战争战场层的公共世界如何组织（多方 Master-Servant 组合在其中的碰撞机制）
- 令咒机制在此架构下的具体触发条件和视觉/情感峰值
- 潜入/入侵玩法的具体接口

## 4. Wow Moment 1 · 召唤仪式与据点空间蜕变

### 4.1 核心画面（一句话）

**从一件普通物品到一个独一无二的英灵世界——AI 生成世界能力的直接证据**。

### 4.2 分段镜头骨架

**段 1 · 召唤仪式（观众能看见 AI 生成过程）**

- 玩家将现实中的普通物品放上魔法阵（观众清楚看得见是普通物品——一枚戒指、一本旧书、一柄餐刀之类）
- 物品浮起，散发光。**符文在其表面自浮现**——AI 从物品的图像/描述实时生成对应的符号
- 玩家周围空间开始扭曲/展开——现实场景后退，新的"英灵背景空间"从中心向外具象化
- 玩家的视角从"看着仪式"切换为"处于新空间之中"
- 英灵身影从空间深处走出/浮现，说出招牌台词

**段 2 · 初遇时的灰模空间**

- 空间**极小**、**基础设施最简**、**颜色近乎灰模**
- 细节隐去、特征模糊、背景 NPC 不存在
- 英灵是空间中唯一清晰的存在

**段 3 · 线索浮现与英灵记忆亲历（Animus 内嵌模块）**

- 随着玩家与英灵的互动/共同行动 → **"线索"浮现在空间里**（一件破损的物品、一段模糊的旋律、一个看不清脸的影子）
- 玩家激活线索 → **视角切换为"以英灵视角亲身经历一段记忆"**（Animus 架构在此处被吸收作为内嵌模块——demo 里可演示这一段第一人称亲历，如玩家在马其顿平原策马冲锋并被军团高呼名字）
- 经历完后：
  - 英灵**获得新的能力**（原作里"了解 Servant = 更能发挥其战力"的机制体现）
  - 空间**解锁新的功能区域**
  - 部分**细节、特征元素、背景 NPC** 显现

**段 4 · 快进：多个线索被激活**

- 蒙太奇形式：数次线索激活 → 数次记忆亲历 → 空间逐步丰盈
- 玩家和英灵之间的**了解累积**（不是通过对白，是通过共同亲历）

**段 5 · 真名揭示 · 空间质变**

- 某个时刻，玩家通过某种方式（对话、供品、共同战斗到某种深度）触发"真名条件"
- **空间从"暗示态"整体转为"命名态"**：
  - 原本模糊的 NPC → 清晰命名（有脸、有名字）
  - 原本无字的物件 → 具体命名（军旗上的题词、地图上的地名、酒杯的来历）
  - 原本封闭的空间中心区 → **大门打开**
  - **原本不存在的地平线区域浮现**（英灵毕生梦想却未达的方向）
- BGM 从环境音 → 主题旋律
- 英灵直视玩家说出真名
- 空间进入"英灵专属世界"完全形态

### 4.3 设计要点（关键约束）

- **物品必须"普通"**：demo 演示用的物品要让观众看得出"这是随便一件东西"，才能证明 AI 的生成能力
- **灰模起点 → 命名态终点 的对比要极强**：这个视觉对比本身就是 wow 的核心冲击
- **记忆亲历段用第一人称**：不是"看动画"，是"是这个英灵"——原作里 Shirou 通过梦体验 Saber 的加冕这一情境的**参与化版本**
- **真名揭示时的"整体命名"是 vision pitch 中最长的单一 wow moment 候选**（约 **8-10 秒** · 在 Wow 1 总时长 30-35 秒中约占 1/4）
- **示范角色**：本段用**知名角色**（如 Iskandar）—— 观众能立刻识别"AI 演对了熟悉角色"

### 4.4 已否掉的方向

- ❌ **不用 Reality Marble 作为 franchise-level 硬绑定**——原作里 Reality Marble 是 EX 级稀有能力，只有极少数 Servant 有。改用**"灵基空间 / 英灵之座的显影"**这类原创说法，即使普通 Servant 也可有独一无二的居所空间

## 5. Wow Moment 2 · 一天时间线与上线并肩作战

### 5.1 核心画面（一句话）

**玩家和英灵在同一战场里，实时对话，实时战术协商，一起打赢一场战斗——过往 Fate 游戏做不到、Character.ai / 乙游结构上不可能演的独家画面**。

### 5.2 分段镜头骨架

**段 1 · 早上 · 一句约定（约 3-5 秒）**

- Master 起床，看手机
- Iskandar 一条消息：*"今晚 20:00。Rider 据点。"*
- Master 回：*"好。"*
- **没多余对话**

**段 2 · 现实的一天 · 快速蒙太奇（约 5-8 秒）**

- Master 上班 / 通勤 / 开会 / 下班的快速切换
- 这一天手机震动**只有一次**：
  - *Iskandar 15:23：侦查完毕。据点 3 条路。晚上你选。*
  - *Master：看到了。*
- **观众看到**：这一天英灵在**为晚上准备**，Master 的现实生活**没被打扰**——**成年人有默契的搭档关系**

**段 3 · 上线 · 并肩作战（wow moment 核心 · 约 15-20 秒）**

- **19:59** 时钟走到。Master 冲进书房，脱外套，打开电脑，戴上耳机，Enter
- 画面切进战场空间。夜里的城市。远处有火光
- Master 的 avatar 站在巷子里
- **Iskandar 就在旁边**——直接说话（战场里的实时语音，不是打字）：
  > *"Master，你终于来了。"*
- Iskandar 转身指向前方建筑：
  > *"Rider 就在那里面。他有一发对军宝具，吟唱时间 3 秒。*
  > *我从右翼冲进去吸引他。你从左翼绕后。*
  > *他开始吟唱的时候——你必须在 3 秒内打断他。*
  > *一次机会。"*
- Master 说：*"我做得到。"*
- Iskandar 点头，冲向建筑右翼
- **战斗开始**：
  - Iskandar 冲进正门
  - 玩家操控 avatar 从左侧墙翻进去
  - Rider 察觉，转向 Iskandar
  - Rider 举起武器，开始吟唱
  - **红色光辉从他身上升起**（宝具展开前奏）
  - Master 从 Rider 背后冲出
  - **一次战术输入**（QTE / 精准操作）
  - Master 击中 Rider 咽喉 → **吟唱中断，宝具消散**
  - Iskandar 一步冲上，一击
  - Rider 消失成金光
- Iskandar 转身看向 Master：*"干得漂亮。"*
- Master 站在原地喘气

**段 4 · 下线交接（约 3 秒）**

- Iskandar 环顾战场：*"今晚 Rider 报废。我留守据点。明天见。"*
- Master 点头。画面切回现实——Master 摘下耳机，靠在椅背上，笑了
- 手机震动一次：*Iskandar 22:14：在整理战果。明早给你。*
- **观众看到**：Master 下线了，但英灵**在异世界持续行动**

**段 5 · 收尾字幕**

> **这不是聊天。这是并肩作战。**
> **你的英灵不睡觉。你睡了他还在打。**
> **你 8 点上线，他从 3 秒前就在等你。**
> **和现实世界同步的、在另一个世界持续进行的、真实的圣杯战争。**

### 5.3 设计要点（关键约束）

- **主戏是并肩作战段（段 3）**，其他段是铺垫。**段 3 时长约占 Wow 2 总时长的一半（15-20 秒 / 30-35 秒）**
- **段 3 里 Iskandar 的战术协商台词是这一整个 vision pitch 的皇冠时刻之一**：*"3 秒。你打断他一次，我就能杀他一次。"* 这一句同时展示：
  - 英灵的独立判断力（自己规划战术）
  - 英灵的主动沟通（清晰告诉玩家要做什么）
  - 英灵的可靠（"我杀他"——英灵承担主要击杀）
  - 玩家的角色（不是发号施令，是关键窗口的战术执行者）
  - 真正的并肩（两个人的角色缺一不可）
- **实时性至关重要**：不是回合制，不是可暂停，不是"选择菜单"。战术协商发生在 3 秒的时间窗里
- **英灵在战斗中主要靠动作和短促台词表达**，不做长对话
- **示范角色**：本段用**同一个知名角色**（继续用 Iskandar，与 Wow 1 保持角色连续性）

### 5.4 已否掉的方向

- ❌ **不演不同英灵与 Master 相处的差异化对比蒙太奇**——上一阶段试过，多个英灵消息对比会被解读为"聊天游戏"
- ❌ **不演玩家学习曲线**——试过"Iskandar 让玩家做后勤 / Arturia 让玩家承担决策 / Hassan 让玩家学会不问"这种"关系错误 → 挫败 → 学习 → 调整"结构，被判定为"演出游戏体验差"
- ❌ **离线消息段不能成为主戏**——离线消息只做"约定"和"衔接"两个功能，多了会退化为聊天游戏
- ❌ **英灵通过独白展现人设**——所有性格表达通过行动、决策、沟通频率、反应模式传达。不写"感人对话"

## 6. Wow Moment 3 · 原创英灵召唤与收尾情感冲击

### 6.1 核心画面（一句话）

**从一件私人物品和一句话，AI 现场召唤出一个不存在于任何 Fate 作品的英灵——AI 生成新角色能力的直接证据 + "每场圣杯战争都是独一无二"的最强情感收尾**。

### 6.2 分段镜头骨架

**段 1 · 圣遗物与描述输入**

- Master 拿起手边一件**普通私人物品**（示范用：一枚很旧的怀表，链子磨白了）
- 放上魔法阵
- 键盘上敲入：*"这是我爷爷的怀表。他在一战里当过工兵。"*

**段 2 · AI 生成过程可视化**

- 屏幕上简洁的文字流：*生成中——匹配英灵之座——找到候选——匹配完成*
- **不必是花哨的动画**——文字化的过程本身就足够（"这是 AI 实时做的，不是预制的"）

**段 3 · 空间展开与英灵登场**

- 空间展开——**观众绝对不认识的场景**：
  - 一条战壕、泥泞、工兵铲、褪色的军旗、远处炮火隐约
- 一个身影从战壕深处走出——**观众绝对不认识的军人**：
  - 中年、有胡茬、一战时期英国工兵制服
- 他看到 Master，敬礼：
  > *"Sir. Corporal [观众看到的名字]. Reporting in."*

**段 4 · 一天时间线（Wow 2 的原创英灵版本，极度压缩 · 约 5-7 秒）**

- 早上：*Sir. Reporting in.*
- 通勤：*Sir. Northern sector reconnaissance complete. Two enemies. Awaiting orders.*
- 下午：*Sir. Engaged. Minor casualty. Combat continues.*
- 深夜：*Sir. Task complete. Reporting for next command.*
- **观众看到**：这个 AI 生成的原创英灵**已经在圣杯战争里活了下来**，有自己的战友日常

**段 5 · 收尾字幕（vision pitch 的最强情感收尾）**

> **这个战友不在任何 Fate 作品里。**
> **他是从 [Master 名字] 爷爷的怀表里被召唤出来的。**
> **和 [Master] 已经共同打了 47 天圣杯战争。**
> **只有 [Master] 一个人知道该怎么和他配合。**
>
> **每一场圣杯战争，都是独一无二的战友关系。**

紧接着 tagline 收尾（见第 1 章）。

### 6.3 设计要点（关键约束）

- **AI 生成过程要"看得见"但不能"看得花哨"**：简洁的文字流最有说服力，"这不是特效，是 AI 在做事"
- **英灵的身份要"陌生但真实"**：一战工兵是一个**观众能立刻识别时代背景、但绝对不是 Fate 系列已知角色**的选择。这个平衡至关重要
- **段 4 的一天时间线是 Wow 2 的原创英灵版本，且必须与 Wow 2 的 Iskandar 版本**风格不同**——Iskandar 是王者语气，工兵是军人语气**——这个对比暗示"每个英灵都独一无二"
- **收尾字幕的"47 天"数字是一个具体锚点**：暗示这不是一次性演示，是一个玩家的持续战友关系
- **"只有 [Master] 一个人知道该怎么和他配合"** 是差异化的**收尾隐性表达**——不需要多个英灵对比蒙太奇也能兑现

### 6.4 已否掉的方向

- ❌ **不用知名 Fate 角色演本段**——Wow 3 的核心是"AI 凭空生成新角色"的证据，用知名角色会破坏这个证据链
- ❌ **不做花哨的召唤特效动画**——文字流 + 空间实体化就够，特效反而会削弱"这是 AI 实时做的"的可信度
- ❌ **不用女性角色或明显二次元审美**——避免被误认为乙游赛道。一战工兵这类**成年人的、历史真实感的**选择是刻意的

## 7. 三个 Wow Moment 之间的结构关系

| 段 | 用什么角色 | 兑现什么核心证据 |
|---|---|---|
| **Wow 1** 召唤+空间+记忆+真名 | **知名角色**（Iskandar 示范） | "AI 演对了熟悉角色" + "AI 生成独一无二的英灵世界" |
| **Wow 2** 一天时间线+并肩作战 | **同一个知名角色**（继续 Iskandar） | "英灵是真的在异世界打仗的战友" + "现实-游戏时间同步" |
| **Wow 3** 原创英灵召唤+收尾 | **AI 现场生成的原创角色**（一战工兵示范） | "AI 凭空生成新角色" + "每场战争独一无二" |

**结构逻辑**：
- **Wow 1 + Wow 2 用同一个知名角色**——让观众在两段之间建立熟悉感，感受"这个 Iskandar 从头到尾是一致的、可信的"
- **Wow 3 切换到原创角色**——制造"这不只是演知名角色，AI 是真的能生成新角色"的证据反差
- **顺序不可颠倒**：先建立熟悉度信任，再展示凭空生成能力

**tagline 只在 Wow 3 结尾出现一次**——不重复，作为整套 vision pitch 的最终收束。

## 8. 3 个必现证据时刻的对照兑现

来自 project-background.md 的 3 个必现证据时刻要求，本 Wow Moment 组合的兑现方式：

| 必现证据 | 兑现在哪段 |
|---|---|
| ① 个人种子 → 世界在眼前生成 | **Wow 1 段 1**（物品 → 空间展开）+ **Wow 3 段 3**（怀表 → 一战工兵战壕） |
| ② AI agent 对无法预演的输入作出活反应 | **Wow 1 段 3**（记忆亲历中英灵的反应）+ **Wow 2 段 3**（并肩作战中英灵的战术协商）+ **Wow 3 段 3**（原创英灵的自主行为） |
| ③ 拉远看到不止这一个世界 | **Wow 3 段 3-5**（原创英灵与 Wow 1 Iskandar 的空间对比 —— **马其顿军营 vs 一战战壕**，两个完全不同的世界，暗示"每次召唤都是一个独一无二的世界"） |

**注意**：③ 不通过"演多个玩家的世界并排"来兑现——那会陷入数量话术。**通过 Wow 1 与 Wow 3 两个截然不同的英灵空间的形态对比来暗示"多个世界"**。

## 10. 给下一个 Agent 的具体任务范围

### 建议任务清单（顺序推荐）

1. **完整分镜脚本**：把三个 Wow Moment 的分段镜头骨架细化到镜头级（每个镜头具体的画面、机位、时长、台词）
2. **总时长节奏**：三个 Wow Moment 加起来在 **90 秒** vision pitch 视频里的时长分配（推荐分配见章节 1.5 · **严格执行，不得突破 90 秒总长**）
3. **视觉参考清单**：为每一段找出 3-5 个视觉参考（电影镜头 / 游戏画面 / 概念图）作为美术方向锚点
4. **AI 生成规格文档**：为 demo 里 AI 需要现场生成的内容（英灵空间、原创英灵形象、战场情境响应等）明确输入格式、输出预期、fallback 处理规则
5. **台词逐句打磨**：Iskandar 的关键台词（"3 秒。你打断他一次，我就能杀他一次"等）和一战工兵的语气打磨
6. **原创英灵的示范角色打磨**：选定一战工兵作为 Wow 3 示范角色的完整背景资料（生平、动机、战术风格、可选择的宝具形态等）——注意这是**示范**而非**必用**，具体 demo 时的物品和描述可能会调整

### 边界

- **不做玩法机制设计** —— 圣杯战争的战斗规则、令咒的具体使用规则、多方 Master 对战规则等**都不是本文档要求下一个 agent 做的**。这些留给后续机制设计阶段
- **不动 tagline 和红线** —— 第 1 章 tagline 和第 2 章红线原则**不可修改**。如果具体内容设计过程中发现红线与内容冲突，**要停下来提问，不要单方面修改红线**
- **不扩充英灵角色数量** —— demo 只演 2 个英灵（Wow 1+2 一个知名 + Wow 3 一个原创）。不加第三个英灵、不加多 Master 对战场景

## 11. 未决事项 / Open Questions

以下事项**暂挂**（对本文档定型的三个 Wow Moment 不构成阻塞，但下一个 agent 或后续会议需要解决）：

1. **示范用的知名英灵是否确定为 Iskandar**：目前是设计中的默认示范角色。原因：粉丝辨识度高、性格戏剧张力强、王者语气便于与 Wow 3 军人语气形成对比。可能的备选：Arturia（更高全球辨识度）。**建议下一个 agent 尝试 Iskandar 作为主案，同时准备 Arturia 备案供选择**
2. **示范用的原创英灵是否确定为一战工兵**：目前的选择理由：观众能识别历史背景（避免文化陌生感）+ 非 Fate 已有角色（保证原创证据）+ 军人语气与 Iskandar 王者语气对比明显。**建议下一个 agent 探索 2-3 个备选原创角色类型**
3. **空间的持久性形态**（永久 / 契约期间 / 半持久）：不影响 Wow Moment 的画面呈现，但影响后续机制设计
4. **潜入/入侵敌方据点空间的玩法接口**：本 vision pitch 不演，但 project-background 里保留了这个玩法方向的口子
5. **战场层的公共世界如何组织**：本 vision pitch 只需要一个"战场里的 Rider 据点"作为 Wow 2 场景，不涉及多方 Master-Servant 在公共战场同时博弈的呈现方式
6. **令咒机制的视觉/情感峰值**：本 Wow Moment 组合里没有令咒关键使用场景（避开了 R2 报告 M3 卖点），但令咒是 Fate 系列核心卖点之一，是否要单独设计一个令咒 wow moment 补充**待后续会议决**
